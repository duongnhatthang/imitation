**Part B: Create `experiments/atari_smoke.py`**

This is a standalone Python script (NOT a Sacred experiment) that runs all three methods on 2 Atari games. It directly uses the imitation library APIs rather than Sacred to keep it simple and debuggable.

The script should:

1. Parse optional CLI args: `--games` (default: "Pong,Breakout"), `--seed` (default: 0), `--n-rounds` (default: 3), `--total-timesteps` (default: 50000), `--n-envs` (default: 8)
2. Load cached random baselines from `experiments/baselines/atari_random_scores.pkl`
3. For each game:
   a. Create training venv (make_atari_training_venv) and eval venv (make_atari_eval_venv)
   b. Load expert policy (load_atari_expert)
   c. Assert obs space match: `assert venv.observation_space == expert.observation_space`
   d. Collect expert score: evaluate expert on eval venv using `rollout.rollout()` with `min_episodes=10`, compute `rollout_stats()["return_mean"]`
   e. Run BC:
      - Create BC trainer with `observation_space`, `action_space`, `rng`, CnnPolicy
      - Collect expert demonstrations using `rollout.rollout()` with `min_episodes=20`
      - Set BC demonstrations, train with `n_epochs=4`
      - Evaluate on eval venv, compute normalized score
   f. Run DAgger:
      - Create fresh BC trainer (new CnnPolicy)
      - Create `SimpleDAggerTrainer` with `venv`, `scratch_dir`, `expert_policy`, `bc_trainer`, `rng`
      - Train with `total_timesteps` and `bc_train_kwargs=dict(n_epochs=4)`
      - Evaluate on eval venv, compute normalized score
   g. Run FTRL:
      - Create fresh BC trainer (new CnnPolicy)
      - Create `FTRLDAggerTrainer` with `alpha=1.0` and same args as DAgger
      - Train with `total_timesteps` and `bc_train_kwargs=dict(n_epochs=4)`
      - Evaluate on eval venv, compute normalized score
   h. Close all venvs
4. Print summary table:
   ```
   === Atari Smoke Test Results ===
   Game       | BC     | DAgger | FTRL   | Expert | Random
   Pong       | 0.XX   | 0.XX   | 0.XX   | XXX.X  | -XX.X
   Breakout   | 0.XX   | 0.XX   | 0.XX   | XXX.X  | X.X
   ```

IMPORTANT implementation notes:
- Use `from stable_baselines3.common.policies import CnnPolicy` (NOT MlpPolicy) for Atari since obs is image-based
- For BC, need to pass `policy=CnnPolicy` or let BC auto-detect from observation space
- BC needs demonstrations as `TransitionsWithRew` or `Transitions`. Use `rollout.flatten_trajectories()` to convert trajectories to transitions
- For DAgger/FTRL scratch_dir, use `tempfile.mkdtemp()`
- Use `imitation.data.rollout.rollout()` for evaluation, NOT `policy_evaluation.eval_policy` (which is a Sacred ingredient)
- The beta_schedule for DAgger should be `None` (uses default linear schedule)
- Each method gets its own fresh BC trainer and CnnPolicy to ensure fair comparison
- For the `rng` parameter, use `np.random.default_rng(seed)`

The script should handle errors gracefully: if one method fails on one game, print the error and continue.
